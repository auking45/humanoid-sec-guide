# Security Vulnerability Checklist

This section presents a security vulnerability checklist, developed based on identified security threats and global cybersecurity standards, regulations, and guidelines, divided into 11 categories. The reference regulations and standards for each category are as follows.

## ① Secure Software Development Framework (SSDF)

This category defines a checklist for developing secure software within a robot system and complies with the **NIST SP 800-218** guide.

> NIST SP 800-218 classifies secure software development practices into the following four practice groups:
> *   **Prepare the Organization (PO)**: Preparing the organization to perform secure software development.
> *   **Protect the Software (PS)**: Protecting the developed software from tampering and unauthorized access.
> *   **Produce Well-Secured Software (PW)**: Developing software with minimal security vulnerabilities.
> *   **Respond to Vulnerabilities (RV)**: Identifying and responding to vulnerabilities in deployed software.

## ② Supply Chain Security

This category defines a checklist applicable to the entire robot system supply chain to counter increasing software supply chain threats, referencing the **NIST SP 800-161** (C-SCRM) guide.

> Following large-scale supply chain security incidents like Log4j and SolarWinds, it has become crucial to systematically identify, assess, and mitigate supply chain risks.

## ③ ~ ⑨ Industrial Control Systems (IEC 62443)

The following seven categories define the basic security requirements for robot systems by referencing **IEC 62443**, an international standard for the cybersecurity of Industrial Control Systems (ICS).

> IEC 62443-1-1 classifies the Foundational Requirements (FR) into the following seven types:
>
> *   **FR 1 - Identification and Authentication (IA)**: Access control requirements to protect against unauthorized access to devices or information.
> *   **FR 2 - Use Control (UC)**: Use control requirements to protect against unauthorized operation of devices or use of information.
> *   **FR 3 - System Integrity (SI)**: Data integrity requirements to protect against unauthorized changes to communication channels.
> *   **FR 4 - Data Confidentiality (DC)**: Data confidentiality requirements for selected communication channels to protect against eavesdropping.
> *   **FR 5 - Restrict Data Flow (RDF)**: Data flow restriction requirements for communication channels to protect against unauthorized information disclosure.
> *   **FR 6 - Timely Response to Event (TRE)**: Requirements for timely detection, reporting, and response to security breaches.
> *   **FR 7 - Resource Availability (RA)**: Resource availability requirements to protect against denial-of-service attacks.

## ⑩ Cyber Resilience

This category defines requirements for maintaining essential functions and enabling rapid recovery after a breach, referencing Europe's **Cyber Resilience Act (CRA)**.

> The CRA is scheduled to be fully implemented in December 2027 and will mandate basic cybersecurity requirements for digital products and software.

## ⑪ Wireless Security

This category defines requirements for robot systems that perform wireless communication, referencing the European Union's (EU) **Radio Equipment Directive (RED)**.

> RED mandates cybersecurity for radio equipment starting from August 2025, requiring all equipment with wireless capabilities sold in the European market to comply with this regulation.
