# 4.2.4. Application Software Security Requirements

These are security requirements to protect the application layer that performs the robot's high-level tasks, responding to the application software threats identified in Chapter 3. The detailed security requirements are as follows.

| Security Requirement | Mitigation Measure | Checklist ID (Vulnerability Check) | Security Solution |
| :--- | :--- | :--- | :--- |
| **Regular Security Checks & Hardening** | Protect against vulnerability-based attacks with regular scanning, security hardening, and disabling unnecessary functions. | `[SI-05]` Input validation<br>`[SI-07]` Error handling<br>`[RA-07]` Security configuration management<br>`[RA-08]` Restriction of unneeded functionality, ports, and services<br>`[WS-20]` Minimization of network interface exposure<br>`[CR-04]` Definition of RTO and RPO<br>`[CR-08]` Establishment of official procedures for reporting and handling security vulnerabilities<br>`[CR-10]` Procedure for improving security measures based on incidents/vulnerabilities | Source Code Analysis Tool,<br>Industrial EDR,<br>Security Certification Consulting |
| **Security Update Policy** | Protect against improper patch management by establishing a regular update policy. | `[SI-10]` Update support verification<br>`[CR-05]` Timely provision and application of security updates and patches<br>`[WS-12]` Update mechanism provision | Patch Management Solution |
| **Supply Chain Security** | Protect against vulnerabilities in open-source/external modules with SBOM management and updates. | `[SC-01]` Manage open source software components<br>`[SC-02]` Manage EoS<br>`[SC-03]` Establish and apply vendor security capability assessment criteria<br>`[SC-04]` Prepare for security impact assessment and response to supply chain changes<br>`[SC-05]` Manage component vulnerabilities<br>`[SC-07]` Implement rapid patch mechanism<br>`[CR-07]` Specification of security update support policy and lifecycle<br>`[WS-19]` Maintenance of up-to-date software and hardware | Source Code Analysis Tool,<br>Patch Management Solution |
| **Authentication-based Access Control** | Protect against unauthorized access and task changes with authentication-based access control and strong authentication policies (e.g., limiting repeated attempts). | `[IA-01]` User identification and authentication<br>`[IA-02]` Device identification and authentication<br>`[IA-03]` User password policy<br>`[IA-06]` Prevention of authenticator feedback<br>`[IA-07]` Limitation on consecutive failed login attempts<br>`[UC-10]` Non-repudiation for users<br>`[CR-13]` Protection of authentication data such as user information<br>`[WS-01]` Wireless user identification and authentication<br>`[WS-02]` Authenticator validity verification<br>`[WS-03]` Authenticator change<br>`[WS-04]` Password strength<br>`[WS-05]` Brute-force protection | Server Access Control Solution |
| **Session Management** | Protect against session abuse with session locking, termination, and concurrent session control. | `[UC-03]` Session lock<br>`[UC-04]` Remote session termination<br>`[UC-05]` Concurrent session control | |
| **Account Management** | Protect against dormant account exploitation with account reviews and deactivation. | `[IA-09]` User account management<br>`[IA-10]` User identifier management<br>`[IA-11]` User authenticator management | Server Access Control Solution |
| **Privilege-based Access Control** | Protect data configuration and status information queries with role-based access control and separation of duties. | `[IA-01]` User identification and authentication<br>`[UC-01]` User authorization<br>`[SI-03]` Security function verification<br>`[WS-06]` Asset access control | Server Access Control Solution,<br>DB Access Control System |
| **Task Notification** | Protect against abnormal task behavior with real-time monitoring and notifications. | `[ER-02]` Continuous monitoring<br>`[UC-06]` Audit log generation<br>`[WS-17]` Log generation<br>`[WS-18]` Inclusion of time information | Industrial Firewall |
| **Event Analysis** | Protect against abnormal behavior with regular log analysis. | `[CR-09]` Securing sufficient logs and supporting security event analysis | |
| **Log Management** | Protect against log tampering with access control, encryption, and rotation policies. | `[SI-03]` Security function verification<br>`[SI-07]` Error handling<br>`[SI-09]` Protection of audit-related information<br>`[UC-06]` Audit log generation<br>`[UC-07]` Audit log storage capacity management<br>`[UC-08]` Response to audit processing failures<br>`[UC-09]` Time control<br>`[CR-09]` Securing sufficient logs and supporting security event analysis<br>`[WS-18]` Inclusion of time information<br>`[WS-23]` Equipment integrity assurance | Backup Management System |
| **Log Traceability** | Protect access and change activities with logs that include time information. | `[UC-06]` Audit log generation<br>`[UC-10]` Non-repudiation for users<br>`[WS-17]` Log generation<br>`[WS-18]` Inclusion of time information | Server Access Control Solution |
| **Input Validation** | Protect against abnormal parameter input with input validation. | `[SI-05]` Input validation | |
| **Integrity Verification** | Protect against data tampering with integrity verification. | `[WS-23]` Equipment integrity assurance<br>`[SI-03]` Security function verification<br>`[SI-04]` Integrity of software and data<br>`[WS-08]` Stored data integrity verification<br>`[WS-10]` Wireless communication data integrity verification<br>`[SC-06]` Verify integrity of robot firmware and software updates | |
| **Data Encryption** | Protect against sensitive data exposure by applying strong cryptographic algorithms. | `[DP-01]` Confidentiality of information<br>`[DP-03]` Use of secure cryptographic mechanisms<br>`[WS-07]` Secure data protection<br>`[WS-14]` Cryptographic adequacy | |
| **Personal Data Protection** | Protect against personal data misuse with minimal collection, encrypted storage, and complete deletion. | `[DP-04]` Personal data protection | Data Erasure Solution |
| **Backup Management** | Protect against data loss with backup and recovery verification. | `[RA-05]` System backup<br>`[CR-01]` Regular backup and management of critical settings/data<br>`[CR-02]` Operation of backup data integrity and recovery verification procedures | Backup Management System |
| **Safe Rollback** | Protect against update failures with an automatic rollback function. | `[CR-06]` Rollback or recovery function in case of update failure | Patch Management Solution |
| **Update Verification** | Protect against the application of tampered updates with signature-based update verification. | `[SC-06]` Verify integrity of robot firmware and software updates<br>`[WS-13]` Wireless network update integrity and authenticity verification | Patch Management Solution |
| **Safe Stop** | Protect against malfunction with an automatic safe mode transition. | `[CR-12]` Implementation of safe stop/mode transition in abnormal states | – |
| **Malware Scanning** | Protect against the exploitation of malicious files with malware scanning. | `[SI-02]` Malware protection | Industrial EDR, Antivirus |
