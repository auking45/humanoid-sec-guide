# 4.2.2. 네트워크 보안 요구사항

로봇 네트워크 통신을 보호하기 위한 보안 요구사항으로, 3장에서 식별된 네트워크 위협에 대응합니다. 세부 보안 요구사항은 다음과 같습니다.

| 보안 요구사항 | 완화 대책 | 체크리스트 번호 (보안 취약점 점검) | 보안 솔루션 |
| :--- | :--- | :--- | :--- |
| **통신 신뢰성 확보** | 변조된 데이터 및 통신을 서명 기반 무결성 검증으로 보호 | `[SI-01]` 통신 신뢰성 보장 | IoT 보안 게이트웨이 |
| **패킷 검증** | 비정상 명령 프레임 및 데이터 패킷을 입력값 유효성 검증으로 보호 | `[SI-05]` 입력값 검증<br>`[SI-07]` 오류 처리<br>`[WS-22]` 외부 입력 관리 | 산업용 IPS,<br>산업용 방화벽,<br>산업용 IDS,<br>IoT 보안 게이트웨이 |
| **서비스 자원 보호** | 자원 고갈 및 세션 남용을 요청 제한·세션 분리로 보호 | `[RA-01]` 서비스 거부 방지 | 산업용 IPS,<br>산업용 보안 스위치 |
| **불필요 기능 및 포트 제한** | 비인가 접근 및 오남용을 포트·기능 차단 및 화이트리스트 정책으로 보호 | `[RA-08]` 불필요한 기능, 포트 및 서비스 제한<br>`[WS-21]` 외부 물리 인터페이스 제한 | 산업용 IDS,<br>산업용 IPS |
| **통신 암호화** | 네트워크 구간의 데이터 도청을 전송구간 암호화(TLS/SSH 등)로 보호 | `[DP-01]` 정보의 기밀성<br>`[DP-03]` 안전한 암호 메커니즘 사용<br>`[WS-14]` 암호화 적정성<br>`[WS-09]` 무선 통신 데이터 보호 | 보안칩,<br>보안 AP |
| **세션 무결성 검증** | 메시지 재전송 공격을 시퀀스·Nonce 기반 검증으로 보호 | `[SI-08]` 세션의 무결성 보호<br>`[WS-11]` 무선 통신 재전송 공격 방지 구현 | |
| **인증 기반 라우팅** | 비인가자의 라우팅 변경을 사용자·장치 인증 기반 제어로 보호 | `[IA-01]` 사용자 식별 및 인증<br>`[IA-02]` 장치 식별 및 인증<br>`[DFR-01]` 네트워크 기능 지원<br>`[DFR-02]` 프로토콜 필터링 | 산업용 방화벽 |
| **라우팅 내역 관리** | 라우팅 조작 및 오남용을 변경 로그 기록 및 추적 관리로 보호 | `[UC-06]` 감사 로그 생성<br>`[WS-17]` 로그 생성<br>`[WS-18]` 시간 정보 포함 | |
| **장치 인증** | 비인가 장치의 제어명령 송신을 장치 인증으로 보호 | `[IA-02]` 장치 식별 및 인증 | IoT 보안 게이트웨이 |
| **키 관리 강화** | 키·인증서 탈취를 안전한 저장소(보안칩/TPM)와 주기적 키 교체로 보호 | `[IA-04]` 공개키 관리<br>`[WS-14]` 암호화 적정성<br>`[WS-16]` 암호화 키 고유성 | 보안칩,<br>TPM 내장 보드 |
| **인증자 보호** | 인증정보 노출을 민감정보 접근제한 및 즉시 폐기로 보호 | `[IA-06]` 인증 정보 노출 방지<br>`[IA-11]` 사용자 인증자 관리 | 서버 접근 통제 솔루션,<br>IoT보안 게이트웨이 |
