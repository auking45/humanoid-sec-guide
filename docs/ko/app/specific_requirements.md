# 애플리케이션 휴머노이드 특화 보안 요구사항

본 문서는 클라우드 AI(VLM) 통신, 원격 제어 앱, 개인정보 보호 등에 관한 애플리케이션 레벨의 휴머노이드 특화 보안 요구사항을 정의합니다. 기존 KISA 보안 요구사항을 보완하며, 세부 휴머노이드 특화 보안 요구사항은 다음과 같습니다.

| ID | 분류 (Category) | 보안 요구사항 (Item) | 상세 요구사항 (Detailed Requirements) |
| :--- | :--- | :--- | :--- |
| **APP-01** | 명령어 무결성 및 인가 | **Data Integrity (Watermarking)** | 센서 데이터(영상 등) 생성 직후 TEE 내에서 디지털 서명(Watermarking)을 생성하여 패킷에 포함시켜 전송 중 조작을 방지해야 함. |
| **APP-02** | 명령어 무결성 및 인가 | **Command Signing** | 클라우드에서 내려오는 모든 제어 명령에는 서버 측 HSM에서 생성된 디지털 서명이 포함되어야 하며 로봇이 이를 즉시 검증해야 함. |
| **APP-03** | 명령어 무결성 및 인가 | **Remote Attestation** | 해킹된 로봇에 명령이 하달되는 것을 막기 위해, 명령 하달 전 'Remote Attestation'을 통해 로봇의 무결성 상태를 확인하고 위반 시 차단해야 함. |
| **APP-04** | 명령어 무결성 및 인가 | **API Throttling** | 클라우드 계정이 탈취되어 발생하는 플러딩(Flooding) 명령을 방어하기 위해 초당 명령 횟수(Rate Limit)를 제한하고 비정상 패턴 시 세션을 파기해야 함. |
| **APP-05** | VLM 안전성 및 개인정보 보호 | **VLM Drift Detection** | 클라우드 AI 모델(VLM)이 해킹되거나 오작동할 경우를 대비하여, 별도의 'Rule-based Checker'를 두어 물리적으로 위험한 명령은 로봇 실행 전 차단해야 함. |
| **APP-06** | VLM 안전성 및 개인정보 보호 | **Zero-Knowledge (비식별화)** | 프라이버시 보호를 위해 가능할 경우 특징점(Feature Vector)만 추출하여 전송하거나, 전송 전 로컬 애플리케이션 단에서 비식별화(얼굴 마스킹 등)를 수행해야 함. |
| **APP-07** | VLM 안전성 및 개인정보 보호 | **Data Residency** | 수집된 영상/음성 데이터는 개인정보 보호법 준수를 위해 익명화 처리 후 지정된 리전(Region)에만 저장되어야 하며, 보존 기간 경과 후 자동 파기되어야 함. |
