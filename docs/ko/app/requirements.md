# 4.2.4. 응용 소프트웨어 보안 요구사항

로봇의 상위 작업을 수행하는 애플리케이션 계층을 보호하기 위한 보안 요구사항으로, 3장에서 식별된 응용 소프트웨어 위협에 대응합니다. 세부 보안 요구사항은 다음과 같습니다.

| 보안 요구사항 | 완화 대책 | 체크리스트 번호 (보안 취약점 점검) | 보안 솔루션 |
| :--- | :--- | :--- | :--- |
| **정기 보안 점검 및 설정 강화** | 취약점 기반 공격을 정기 스캔·보안 설정 강화·불필요 기능 비활성화로 보호 | `[SI-05]` 입력값 검증<br>`[SI-07]` 오류 처리<br>`[RA-07]` 보안 구성 설정 관리<br>`[RA-08]` 불필요한 기능, 포트 및 서비스 제한<br>`[WS-20]` 네트워크 인터페이스 노출 최소화<br>`[CR-04]` 복구 목표 시간(RTO) 및 복구 목표 시점(RPO) 정의<br>`[CR-08]` 보안 취약점 보고 처리 공식 절차 마련<br>`[CR-10]` 보안 사고/취약점 기반 보안 대책 개선 절차 | 소스코드 진단 도구,<br>산업 EDR,<br>보안인증 컨설팅 |
| **보안 업데이트 정책** | 비적절한 패치 관리를 정기 업데이트 정책 수립으로 보호 | `[SI-10]` 업데이트 지원 검증<br>`[CR-05]` 보안 업데이트 및 패치의 적시 제공과 적용<br>`[WS-12]` 업데이트 메커니즘 제공 | 패치 관리 솔루션 |
| **공급망 보안** | 오픈소스·외부 모듈 취약점을 SBOM 관리 및 최신화로 보호 | `[SC-01]` 오픈소스 소프트웨어 구성요소 관리<br>`[SC-02]` EoS 관리<br>`[SC-03]` 공급업체 보안 역량 평가 기준 수립 및 적용<br>`[SC-04]` 공급망 변경 시 보안 영향 평가 및 대처 방안 마련<br>`[SC-05]` 구성요소 취약점 관리<br>`[SC-07]` 신속한 패치 메커니즘<br>`[CR-07]` 보안 업데이트 지원 정책 및 수명주기 명시<br>`[WS-19]` 최신 소프트웨어 및 하드웨어 유지 | 소스코드 진단 도구,<br>패치 관리 솔루션 |
| **인증 기반 접근 통제** | 비인가 접근 및 작업 변경을 인증 기반 접근 제어와 강력한 인증 정책(반복 시도 제한 등)으로 보호 | `[IA-01]` 사용자 식별 및 인증<br>`[IA-02]` 장치 식별 및 인증<br>`[IA-03]` 사용자 패스워드 정책<br>`[IA-06]` 인증정보 노출 방지<br>`[IA-07]` 반복된 로그인 시도 제한<br>`[UC-10]` 사용자에 대한 부인 방지<br>`[CR-13]` 사용자 정보 등의 인증 데이터 보호<br>`[WS-01]` 무선 사용자 식별 및 인증<br>`[WS-02]` 인증자 유효성 검증<br>`[WS-03]` 인증자 변경<br>`[WS-04]` 비밀번호 강도<br>`[WS-05]` 무차별 대입 보호 | 서버 접근 통제 솔루션 |
| **세션 관리** | 세션 남용을 세션 잠금·종료·동시 접속 제어로 보호 | `[UC-03]` 세션 잠금<br>`[UC-04]` 원격 세션종료<br>`[UC-05]` 동시 세션제어 | |
| **계정 관리** | 휴면 계정 악용을 계정 점검·비활성화로 보호 | `[IA-09]` 사용자 계정 관리<br>`[IA-10]` 사용자 식별자 관리<br>`[IA-11]` 사용자 인증자 관리 | 서버 접근 통제 솔루션 |
| **권한 기반 접근 통제** | 데이터 설정 및 상태 정보 조회를 역할 기반 권한 통제와 권한 분리로 보호 | `[IA-01]` 사용자 식별 및 인증<br>`[UC-01]` 사용자 권한 부여<br>`[SI-03]` 보안 기능 검증<br>`[WS-06]` 자산 접근 제어 | 서버 접근 통제 솔루션,<br>DB 접근 제어 시스템 |
| **작업 알림** | 작업 이상 행위를 실시간 모니터링과 알림으로 보호 | `[ER-02]` 지속적인 모니터링<br>`[UC-06]` 감사 로그 생성<br>`[WS-17]` 로그 생성<br>`[WS-18]` 시간 정보 포함 | 산업용 방화벽 |

| **로그 관리** | 로그 변조를 접근제어·암호화·로테이션 정책으로 보호 | `[SI-03]` 보안 기능 검증<br>`[SI-07]` 오류 처리<br>`[SI-09]` 감사관련 정보 보호<br>`[UC-06]` 감사 로그 생성<br>`[UC-07]` 감사 로그 저장 용량 관리<br>`[UC-08]` 감사 처리 실패 대응<br>`[UC-09]` 시간 통제<br>`[CR-09]` 충분한 로그 확보 및 보안이벤트 분석 지원<br>`[WS-18]` 시간 정보 포함<br>`[WS-23]` 장비 무결성 보장 | 백업 관리 시스템 |
| **로그 추적성** | 접근 및 변경 행위를 시간정보 포함 로그 생성으로 보호 | `[UC-06]` 감사 로그 생성<br>`[UC-10]` 사용자에 대한 부인 방지<br>`[WS-17]` 로그 생성<br>`[WS-18]` 시간 정보 포함 | 서버 접근 통제 솔루션 |
| **입력값 검증** | 비정상 파라미터 입력을 입력값 검증으로 보호 | `[SI-05]` 입력값 검증 | |
| **무결성 검증** | 데이터 변조를 무결성 검증으로 보호 | `[WS-23]` 장비 무결성 보장<br>`[SI-03]` 보안 기능 검증<br>`[SI-04]` 소프트웨어 및 데이터 무결성 검증<br>`[WS-08]` 저장 데이터 무결성 검증<br>`[WS-10]` 무선 통신 데이터 무결성검증<br>`[SC-06]` 로봇 펌웨어 및 소프트웨어 업데이트의 무결성 검증 | |

| **안전 정지** | 오동작을 자동 안전 모드 전환으로 보호 | `[CR-12]` 비정상 상태에서의 안전 정지/모드 전환 구현 | – |
