# 휴머노이드 보안 가이드 (HSG)

이 가이드는 **KISA 로봇 보안 모델 (2025)**을 전면적으로 도입하여 휴머노이드 로봇을 위한 기술적 보안 명세서를 제공합니다. 로봇 시스템의 전체 생애주기를 안전하게 보호하기 위한 포괄적인 프레임워크 역할을 합니다.

## 로봇 서비스 주요 보안 목표 (Security Objectives)

휴머노이드 로봇이 안전하게 서비스를 제공하기 위해 달성해야 할 6대 핵심 보안 목표입니다. KISA 가이드라인에 따라 일반 IT 시스템의 보안 목표(기밀성, 무결성, 가용성)와 로봇 특화 보안 목표(정확성, 신뢰성, 안전성)로 구분됩니다.

### IT 시스템 보안 목표
* **기밀성 (Confidentiality)**: 비인가된 접근으로부터 데이터를 보호하며, 로봇의 센서 정보(영상·음성·위치), 동작 로그, 공정 정보 등 물리 환경 및 운영 정보의 노출을 방지합니다.
* **무결성 (Integrity)**: 비인가된 수정, 변조, 삭제로부터 데이터를 보호하여, 센서 데이터, 제어 명령, 작업 프로그램, 안전 설정의 변조 및 재전송으로 인한 오동작 및 사고를 방지합니다.
* **가용성 (Availability)**: 인가된 사용자가 필요한 시점에 시스템 및 데이터에 안정적으로 접근할 수 있도록 하며, 로봇 작업 중단이 연쇄 안전정지로 확대되지 않도록 운영 연속성과 복구 가능성을 확보합니다.

### 로봇 특화 보안 목표
* **정확성 (Accuracy)**: 로봇의 목표 위치·자세·속도·힘(토크) 등을 허용 오차 범위 내에서 수행하도록 제어 성능을 보장하며, 물리 동작 제어가 오차를 벗어나 발생할 수 있는 충돌 및 품질 저하를 방지합니다.
* **신뢰성 (Reliability)**: 로봇이 동일 조건에서 예측 가능하고 일관된 동작을 수행하며, 상태·경보·진단 정보가 안전한 운영 및 분석에 활용 가능할 수준으로 유지되도록 보장합니다.
* **안전성 (Safety)**: 로봇의 물리 동작이 사람·설비·환경에 위해를 유발하지 않도록 위험을 통제하고, 이상 징후 발생 시 즉시 안전정지/안전상태(Fail-Safe)로 전환되도록 보장합니다.

## 핵심 영역 (Core Domains)

1. **하드웨어 보안 (Hardware Security)**: 하드웨어 신뢰점(RoT), 안전한 저장소, 물리적 훼손 방지 등을 통한 기기 무결성 확보.
2. **네트워크 보안 (Network Security)**: 강력한 암호화 및 네트워크 분리를 통해 로봇, 제어 시스템, 클라우드 플랫폼 간의 통신 채널 보호.
3. **시스템 SW 보안 (System SW Security)**: 보안 부팅, 접근 통제, 지속적인 모니터링을 통해 운영 체제 및 플랫폼 계층 강화.
4. **애플리케이션 보안 (Application Security)**: 안전한 개발 생애주기, 제어 명령 무결성, 개인정보 보호 메커니즘을 통한 로봇 서비스 및 데이터 보호.
